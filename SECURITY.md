# Security Policy

## Reporting a Vulnerability

Please do not open a public issue for security vulnerabilities.

Use GitHub's private vulnerability reporting feature for this repository, if
available. If private reporting is not available, contact the project
maintainers through the official CommitQuest organization channels.

Include:

- A short description of the issue
- Steps to reproduce
- The affected version or commit
- Any relevant logs, screenshots, or proof-of-concept details

Please do not include real user tokens, API keys, or private account data in
reports.

## Supported Versions

Security fixes are prioritized for the latest released version of the CLI.
